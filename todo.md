# 지역별 브랜드 상세 → H-2 디자인(매거진 카드형 아코디언) 적용 작업 — ✅ 완료

## 배경
`timeline-design-options.html`에서 만든 "옵션 H-2" 디자인(제조사 단위로 접고 펼치는 아코디언 + 좌/우 2단 카드)을
실제 `guide.html`의 "지역별 브랜드" 탭 상세 내용(기존 `<table class="card">` 구조)에 적용.

## 결과: 6개 지역 전체 변환 완료

| 지역 | 브랜드 카드 수 | 제조사 그룹 수 |
|---|---|---|
| 서울·경기 | 46 | 10 |
| 충청도 | 18 | 4 |
| 강원도 | 9 | 2 |
| 전라도 | 14 | 5 |
| 경상도 | 67 | 13 |
| 제주도 | 13 | 4 |
| **합계** | **167** | **38** |

원본 테이블 167개 `.cn` 항목, 38개 제조사 테이블과 정확히 일치 — 데이터 유실 없음 (grep으로 최종 검증 완료).

## 적용한 내용
1. **CSS 추가** (`guide.html` `<style>` 섹션, `.ab/.at/.ad/.aa` accent 클래스 블록 바로 뒤):
   `.rgH2-group`/`.rgH2-head`/`.rgH2-count`/`.rgH2-caret`/`.rgH2-body`(제조사 아코디언), `.rgH-card`/`.rgH-left`/`.rgH-right`/
   `.rgH-year`/`.rgH-name`/`.rgH-badges`(브랜드 카드). 모바일(`@media max-width:640px`)·인쇄(`@media print`: 접힌
   아코디언도 강제 펼침) 대응 포함. 기존 `.lbl/.ety/.det`(설명 텍스트), `.abv-badge/.zs-badge/.disc-badge`(배지),
   `.cb-b/.cb-t/.cb-d/.cb-a`(헤더 배경색)는 재사용.
2. **모든 지역 HTML 변환**: `<table class="card">` → `<details class="rgH2-group"><summary class="rgH2-head [cb-색상]">
   제조사명<span class="rgH2-count">N종</span>...</summary><div class="rgH2-body">...카드들...</div></details>`.
   기본 접힌 상태 유지(원래 UX와 동일). 헤더 색상은 제조사가 아니라 원본 테이블의 `cb-*` 클래스를 그대로 따름
   (카테고리 기준 색상이며, 경상도처럼 희석식·플레이버가 `cb-a`(amber)를 쓰는 지역별 예외도 원본 그대로 보존).
3. **JS 수정** (`initRegionSummaries` 함수): 기존 `.cb`/`table.card` 기반 카운팅에 `.rgH2-group`/`.rgH-card` 기반
   카운팅을 병행 추가해, 제조사 수·카테고리별 브랜드 수 집계가 모든 지역에서 정상 동작하도록 함.
   (`initNameBadges`/`initBadgeRows`/`initCardAccordions`는 이제 어느 지역에서도 대상 요소가 없어 자동으로 no-op —
   삭제하지 않고 그대로 둠, 안전.)

## 추가 변경: 카테고리별 타이틀 색상 통일
원본 테이블은 지역마다 카테고리-색상 매칭이 제각각이었음(예: 강원도 희석식이 보라색, 경상도 희석식·플레이버가
둘 다 주황색). 사용자 요청으로 전체 6개 지역에 걸쳐 **희석식=파랑(`cat-diluted`, #1565c0) / 플레이버=주황
(`cat-flavor`, #e65100) / 증류식=초록(`cat-distilled`, #2e7d32)**으로 강제 통일. `cat-subhead` 텍스트를 기준으로
Python 스크립트로 일괄 치환(제조사 헤더 38개, 브랜드 카드 167개 모두 카테고리 개수와 정확히 일치 검증 완료).
기존 `cb-b/cb-t/cb-d/cb-a`, `ab/at/ad/aa` 클래스와 CSS 규칙은 더 이상 사용되지 않지만 죽은 코드로 남겨둠(요청 범위 밖).

## 남은 확인 사항
- 브라우저에서 실제 렌더링(아코디언 펼침/접힘, 배지 표시, 카테고리 색상, 모바일 폭, 제조사/브랜드 개수 요약 칩)을
  육안으로 최종 확인 필요 — 구조적 검증(카운트, 텍스트 유실 여부)은 완료됐으나 시각적 확인은 아직 사용자 몫.
- 문제 발견 시 해당 지역만 부분 수정하면 됨 (지역별로 독립적인 블록이라 서로 영향 없음).
